System Health¶

System Health reports the machine state for the host running Shogun Command. It powers the Mission Control health cards, service watch, notices, and live refresh API responses.

Overview¶

User Scenario¶

User Scenario: Operator checks live host readiness SCN_SYSHEALTH_OPERATOR_CHECKS_HOST
status: active
owner: Diogo
links incoming: REQ_SYSHEALTH_HOST_REPORTED [Functional Requirement], REQ_SYSHEALTH_SERVICES_REPORTED [Functional Requirement], REQ_SYSHEALTH_SAME_ORIGIN_REFRESH [Functional Requirement]

An authenticated operator opens Mission Control and sees current host, memory, disk, thermal, load, and service readiness for the machine running Shogun Command.

Functional Requirements¶

Functional Requirement: Host health is reported REQ_SYSHEALTH_HOST_REPORTED
status: active
owner: Diogo
links outgoing: SCN_SYSHEALTH_OPERATOR_CHECKS_HOST [User Scenario]
links incoming: ARCH_SYSHEALTH_FEATURE_BOUNDARY [Architecture Note], TEST_SYSHEALTH_BUILD_VALIDATION [Validation]

Shogun Command must report host identity, uptime, CPU, memory, disk, thermal, load, status, notices, and the time the check was performed.
Functional Requirement: Services are reported REQ_SYSHEALTH_SERVICES_REPORTED
status: active
owner: Diogo
links outgoing: SCN_SYSHEALTH_OPERATOR_CHECKS_HOST [User Scenario]
links incoming: ARCH_SYSHEALTH_FEATURE_BOUNDARY [Architecture Note], TEST_SYSHEALTH_BUILD_VALIDATION [Validation]

Shogun Command must report configured service names with active state, status text, and diagnostic detail.
Functional Requirement: Health refresh stays same-origin REQ_SYSHEALTH_SAME_ORIGIN_REFRESH
status: active
owner: Diogo
links outgoing: SCN_SYSHEALTH_OPERATOR_CHECKS_HOST [User Scenario]
links incoming: ARCH_SYSHEALTH_FEATURE_BOUNDARY [Architecture Note], IMPL_SYSHEALTH_DASHBOARD_COMPONENT [Implementation Note], TEST_SYSHEALTH_PROTECTED_API [Validation]

Mission Control must refresh system health through protected same-origin API routes.

Architecture¶

Architecture Note: System health feature boundary ARCH_SYSHEALTH_FEATURE_BOUNDARY
status: active
owner: Diogo
links outgoing: REQ_SYSHEALTH_HOST_REPORTED [Functional Requirement], REQ_SYSHEALTH_SERVICES_REPORTED [Functional Requirement], REQ_SYSHEALTH_SAME_ORIGIN_REFRESH [Functional Requirement]
links incoming: IMPL_SYSHEALTH_SERVER_MODULE [Implementation Note]

features/operations/system-health owns the server readers, response types, and Mission Control dashboard component for system health. The Next.js route handlers under app/api/system stay thin and only serialize the feature output.

Implementation¶

Implementation Note: System health server module IMPL_SYSHEALTH_SERVER_MODULE
status: active
owner: Codex
links outgoing: ARCH_SYSHEALTH_FEATURE_BOUNDARY [Architecture Note]

features/operations/system-health/server.ts reads OS state with Node APIs and small host commands. It preserves the existing SystemHealth and ServiceState response shapes.
Implementation Note: System health dashboard component IMPL_SYSHEALTH_DASHBOARD_COMPONENT
status: active
owner: Codex
links outgoing: REQ_SYSHEALTH_SAME_ORIGIN_REFRESH [Functional Requirement]

features/operations/system-health/components/system-dashboard.tsx renders the Mission Control health UI and refreshes /api/system/health and /api/system/services every five seconds.

Validation¶

Validation: System health build validation TEST_SYSHEALTH_BUILD_VALIDATION
status: active
owner: Diogo
links outgoing: REQ_SYSHEALTH_HOST_REPORTED [Functional Requirement], REQ_SYSHEALTH_SERVICES_REPORTED [Functional Requirement]

npm run build should compile the system health feature, API routes, and dashboard imports without changing public response shapes.
Validation: System health protected API validation TEST_SYSHEALTH_PROTECTED_API
status: active
owner: Diogo
links outgoing: REQ_SYSHEALTH_SAME_ORIGIN_REFRESH [Functional Requirement]

An anonymous request to /api/system/health should return 401 JSON, while an authenticated dashboard session should refresh health and services.
